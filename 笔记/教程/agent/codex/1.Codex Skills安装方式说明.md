# Codex Skills 安装方式说明

更新时间：2026-07-14  
本机验证版本：`codex-cli 0.144.3`

## 适用场景

这篇教程用于给 Codex 安装和验证 skills。适合以下情况：

- 想安装第三方 skills，例如 Anthropic skills 仓库里的 `frontend-design`、`webapp-testing`。
- 想使用 OpenAI curated skills，例如通过 Codex 内置 `skill-installer` 安装。
- 想手动放置单个 skill。
- 想确认当前 Codex 到底扫描了哪些 skill 目录。

## 先了解 Codex 会扫描哪些目录

当前这台机器的 Codex 已确认会读取以下 skill root：

```text
C:\Users\lylyu\.codex\skills
C:\Users\lylyu\.agents\skills
C:\Users\lylyu\.codex\skills\.system
```

所以，第三方通用 skills 推荐放在：

```text
C:\Users\<you>\.agents\skills
```

Codex 本地安装或通过 `skill-installer` 安装的 skills 通常放在：

```text
C:\Users\<you>\.codex\skills
```

Codex 系统内置 skills 位于：

```text
C:\Users\<you>\.codex\skills\.system
```

如果某个 skill 只想给单个仓库使用，可以放到仓库内：

```text
<repo>\.agents\skills
```

官方 Codex 文档把 `USER` scope 写成 `$HOME/.agents/skills`，用于个人全局 skills；也写明 Codex 会扫描 repo 内从当前目录到 repo root 的 `.agents/skills`，并且支持 symlinked skill folders。

## 安装 Anthropic skills 仓库

### 1. 创建 skills 目录

PowerShell：

```powershell
New-Item -ItemType Directory -Force "$env:USERPROFILE\.agents\skills"
```

### 2. 克隆 Anthropic skills 仓库

```powershell
git clone https://github.com/anthropics/skills.git "$env:USERPROFILE\.agents\skills\anthropic-skills"
```

安装后目录应类似：

```text
C:\Users\<you>\.agents\skills\anthropic-skills\skills\frontend-design\SKILL.md
C:\Users\<you>\.agents\skills\anthropic-skills\skills\webapp-testing\SKILL.md
```

以这台机器为例，`frontend-design` 的实际路径是：

```text
C:\Users\lylyu\.agents\skills\anthropic-skills\skills\frontend-design\SKILL.md
```

### 3. 重启 Codex 或新开会话

安装完成后，重启 Codex 或新开一个会话，然后用 `/skills` 或在输入框输入 `$` 查看 skill 是否出现。

如果新会话里能看到 `frontend-design`，说明 `.agents\skills` 已经被当前 Codex 客户端扫描。

### 4. 更新 Anthropic skills

后续更新时执行：

```powershell
git -C "$env:USERPROFILE\.agents\skills\anthropic-skills" pull
```

## 安装 OpenAI curated skills

OpenAI curated skills 推荐在 Codex 对话框里使用内置 `skill-installer`，不要在 PowerShell 里直接输入 `$skill-installer`。

在 Codex 对话框输入：

```text
$skill-installer playwright
```

也可以直接告诉 Codex：

```text
帮我安装 playwright skill
```

注意：`$skill-installer` 是 Codex 的 skill 调用语法，不是 PowerShell 命令。PowerShell 中 `$xxx` 是变量语法，所以会报类似：

```text
Unexpected token '-installer' in expression or statement.
```

## 手动安装单个第三方 skill

如果不想 clone 整个仓库，可以把单个 skill 目录放到 `.agents\skills` 下。目录必须包含 `SKILL.md`。

例如：

```text
C:\Users\<you>\.agents\skills\webapp-testing\SKILL.md
```

这台机器上，之前也用 `skill-installer` 把 `webapp-testing` 安装到了：

```text
C:\Users\lylyu\.codex\skills\webapp-testing
```

这同样能被 Codex 发现，因为 `.codex\skills` 也是当前 Codex 的扫描目录之一。

## 多个工具共用 skills：使用 Junction

如果多个 agent 工具扫描不同目录，可以用 Junction 让它们共享同一份 skill 文件。

例子：

```powershell
cmd /c mklink /J "%USERPROFILE%\.codex\skills\frontend-design" "%USERPROFILE%\.agents\skills\anthropic-skills\skills\frontend-design"
```

对当前 Codex 版本来说，`.agents\skills` 已经会被扫描，所以 Anthropic skills 仓库不需要额外 Junction。

已有的 `superpowers` Junction 是另一种方向：

```text
C:\Users\lylyu\.agents\skills\superpowers
  -> C:\Users\lylyu\.codex\superpowers\skills
```

这里表示真实源在 `.codex\superpowers\skills`，`.agents` 只是兼容入口。Junction 方向没有绝对对错，取决于哪个目录负责维护和更新。

## 验证安装结果

### 查看 Codex 版本

```powershell
codex --version
```

### 检查 skill 文件是否存在

例如检查 Anthropic skills 中的 `frontend-design`：

```powershell
Test-Path "$env:USERPROFILE\.agents\skills\anthropic-skills\skills\frontend-design\SKILL.md"
```

返回 `True` 表示文件存在。

### 检查 Codex 会话里是否暴露 skills

新开 Codex 会话后输入：

```text
/skills
```

或在输入框输入：

```text
$
```

如果能看到安装的 skill 名称，说明 Codex 已经识别到该 skill。

### 从 session 记录里核对 skill roots

如果要从 session 记录里核对 Codex 实际注入了哪些 skill roots，可以在：

```text
C:\Users\<you>\.codex\sessions\...
```

搜索：

```text
Skill roots
.agents/skills
frontend-design
```

## 版本兼容提示

不同 Codex 版本对 skill root 的支持可能不同：

- PR `#10317` 支持 repo scope 的 `.agents/skills`。
- Issue `#10430` 显示 `codex-cli 0.94.0` 时用户全局 `~/.agents/skills` 还没有按预期加载。
- Issue `#10493` 请求默认支持用户全局 `~/.agents/skills`。
- 本机当前 `codex-cli 0.144.3` 已确认会把 `C:\Users\lylyu\.agents\skills` 作为 skill root 注入会话。

因此，本文步骤适用于当前环境。旧 Codex 版本或不同发行渠道可能表现不同，安装后应使用 `/skills` 或 session 记录确认。
