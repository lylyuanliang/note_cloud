# 麒麟 V10 后端中间件手动安装教程

> 本文由服务器环境安装记录整理而来，已移除或替换服务器 IP、账号、密码、内部项目名、内部截图链接等敏感信息。实际使用时请将占位符替换为自己的环境值。

## 1.环境基准

本文基于以下环境整理：

- 操作系统：麒麟 V10
- 部署方式：Linux 原生安装
- CPU 架构：请根据实际服务器选择 x86_64 或 aarch64 安装包
- 适用场景：开发、测试、演示环境
- 生产说明：生产环境需要补充高可用、备份、监控、权限收敛和安全审计

## 2.适用场景

本文适用于不使用 Docker，直接在 Linux 服务器上部署后端项目依赖的场景。覆盖内容包括：

- JDK 17
- MySQL 5.7
- Redis
- Kafka KRaft 单机多实例
- Nacos
- Nginx
- Canal
- 防火墙与后端连接信息整理

## 3.占位符说明

| 占位符 | 含义 |
| --- | --- |
| `<SERVER_IP>` | 当前部署服务器 IP |
| `<CLIENT_IP>` | 允许访问中间件的客户端 IP |
| `<MYSQL_ROOT_PASSWORD>` | MySQL root 密码 |
| `<MYSQL_APP_USER>` | 后端业务 MySQL 用户 |
| `<MYSQL_APP_PASSWORD>` | 后端业务 MySQL 密码 |
| `<REDIS_PASSWORD>` | Redis 密码 |
| `<NACOS_PASSWORD>` | Nacos 密码 |
| `<CANAL_PASSWORD>` | Canal 连接 MySQL 的密码 |
| `<APP_SCHEMA_DEV>` | 开发环境业务库名 |
| `<APP_SCHEMA_TEST>` | 测试环境业务库名 |
| `<APP_NAME>` | 项目或应用名称 |

## 4.目录规划

建议统一把安装包、应用、数据、日志和脚本分开，后续迁移和排障会更清晰。

```bash
sudo mkdir -p /data/softwares
sudo mkdir -p /data/apps
sudo mkdir -p /data/data
sudo mkdir -p /data/logs
sudo mkdir -p /data/scripts
```

目录用途：

| 目录 | 用途 |
| --- | --- |
| `/data/softwares` | 存放 tar.gz、zip 等安装包 |
| `/data/apps` | 存放解压后的应用程序 |
| `/data/data` | 存放 MySQL、Redis、Kafka 等数据文件 |
| `/data/logs` | 存放服务日志 |
| `/data/scripts` | 存放启动、停止、备份等脚本 |

## 5.JDK 17

### 5.1 下载并解压

根据服务器架构选择 JDK 包，示例使用 BiSheng JDK 17：

```bash
cd /data/softwares
uname -m

# x86_64 示例
wget https://mirrors.huaweicloud.com/kunpeng/archive/compiler/bisheng_jdk/bisheng-jdk-17.0.11-linux-x64.tar.gz

sudo tar -zxvf bisheng-jdk-17.0.11-linux-x64.tar.gz -C /data/apps
sudo ln -s /data/apps/bisheng-jdk-17.0.11 /data/apps/jdk17
```

### 5.2 配置环境变量

```bash
sudo tee /etc/profile.d/jdk17.sh >/dev/null <<'EOF'
export JAVA_HOME=/data/apps/jdk17
export PATH=$JAVA_HOME/bin:$PATH
EOF

source /etc/profile.d/jdk17.sh
java -version
javac -version
```

如果服务器上存在多个 JDK，可以使用 `alternatives` 管理：

```bash
sudo alternatives --install /usr/bin/java java /data/apps/jdk17/bin/java 1700
sudo alternatives --install /usr/bin/javac javac /data/apps/jdk17/bin/javac 1700
sudo alternatives --config java
sudo alternatives --config javac
```

## 6.MySQL 5.7

### 6.1 清理冲突包

```bash
rpm -qa | grep -i mariadb
sudo yum remove -y mariadb-libs
sudo rm -rf /var/lib/mysql /etc/my.cnf
```

### 6.2 下载并安装

```bash
cd /data/softwares
wget https://cdn.mysql.com/archives/mysql-5.7/mysql-5.7.44-linux-glibc2.12-x86_64.tar.gz

sudo tar -zxvf mysql-5.7.44-linux-glibc2.12-x86_64.tar.gz -C /data/apps
sudo ln -s /data/apps/mysql-5.7.44-linux-glibc2.12-x86_64 /data/apps/mysql

sudo groupadd mysql
sudo useradd -r -g mysql -s /bin/false mysql
sudo mkdir -p /data/data/mysql /data/logs/mysql
sudo chown -R mysql:mysql /data/apps/mysql-5.7.44-linux-glibc2.12-x86_64 /data/data/mysql /data/logs/mysql
```

### 6.3 配置 my.cnf

```ini
[mysqld]
basedir=/data/apps/mysql
datadir=/data/data/mysql
socket=/tmp/mysql.sock
pid-file=/data/data/mysql/mysql.pid
log-error=/data/logs/mysql/error.log

character-set-server=utf8mb4
collation-server=utf8mb4_unicode_ci

bind-address=0.0.0.0
port=3306
max_connections=1000

default-time-zone=+8:00
explicit_defaults_for_timestamp=true
lower_case_table_names=1

[client]
default-character-set=utf8mb4
socket=/tmp/mysql.sock
```

### 6.4 初始化并启动

```bash
sudo /data/apps/mysql/bin/mysqld --initialize --user=mysql --basedir=/data/apps/mysql --datadir=/data/data/mysql

# 从错误日志中查看临时 root 密码，示例中不要记录真实密码到文档
sudo grep 'temporary password' /data/logs/mysql/error.log

sudo cp /data/apps/mysql/support-files/mysql.server /etc/init.d/mysqld
sudo chmod +x /etc/init.d/mysqld
sudo service mysqld start
```

首次登录后修改 root 密码：

```sql
ALTER USER 'root'@'localhost' IDENTIFIED BY '<MYSQL_ROOT_PASSWORD>';
FLUSH PRIVILEGES;
```

### 6.5 创建业务用户

建议只给业务库授权，不要直接授予全库权限。

```sql
CREATE DATABASE `<APP_SCHEMA_DEV>` DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE `<APP_SCHEMA_TEST>` DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

CREATE USER '<MYSQL_APP_USER>'@'%' IDENTIFIED BY '<MYSQL_APP_PASSWORD>';
GRANT ALL PRIVILEGES ON `<APP_SCHEMA_DEV>`.* TO '<MYSQL_APP_USER>'@'%';
GRANT ALL PRIVILEGES ON `<APP_SCHEMA_TEST>`.* TO '<MYSQL_APP_USER>'@'%';
FLUSH PRIVILEGES;
```

如果可以固定客户端 IP，推荐改成：

```sql
CREATE USER '<MYSQL_APP_USER>'@'<CLIENT_IP>' IDENTIFIED BY '<MYSQL_APP_PASSWORD>';
GRANT ALL PRIVILEGES ON `<APP_SCHEMA_DEV>`.* TO '<MYSQL_APP_USER>'@'<CLIENT_IP>';
FLUSH PRIVILEGES;
```

## 7.Redis

### 7.1 安装

```bash
sudo yum install -y redis
redis-cli --version
```

### 7.2 配置

常见配置文件路径为 `/etc/redis.conf`：

```conf
dir /data/data/redis
logfile /data/logs/redis/redis.log
requirepass <REDIS_PASSWORD>
bind 0.0.0.0
protected-mode yes
maxmemory 256mb
maxmemory-policy allkeys-lru
```

创建目录并授权：

```bash
sudo mkdir -p /data/data/redis /data/logs/redis
sudo chown -R redis:redis /data/data/redis /data/logs/redis
```

启动并设置开机自启：

```bash
sudo systemctl daemon-reload
sudo systemctl enable redis
sudo systemctl start redis
sudo systemctl status redis
```

验证：

```bash
redis-cli -a '<REDIS_PASSWORD>' ping
```

## 8.Kafka KRaft

Kafka 3.x 可以使用 KRaft 模式，不再依赖 ZooKeeper。单机部署时至少需要一个 broker/controller 进程；如果同一台服务器要区分开发、测试、压测环境，需要使用不同端口和数据目录。

### 8.1 下载并解压

```bash
cd /data/softwares
wget https://archive.apache.org/dist/kafka/3.9.0/kafka_2.13-3.9.0.tgz
sudo tar -zxvf kafka_2.13-3.9.0.tgz -C /data/apps
sudo ln -s /data/apps/kafka_2.13-3.9.0 /data/apps/kafka
sudo mkdir -p /data/data/kafka /data/logs/kafka
```

### 8.2 单实例配置

编辑 `/data/apps/kafka/config/kraft/server.properties`：

```properties
process.roles=broker,controller
node.id=1
controller.listener.names=CONTROLLER
controller.quorum.voters=1@127.0.0.1:9093

listeners=PLAINTEXT://0.0.0.0:9092,CONTROLLER://127.0.0.1:9093
advertised.listeners=PLAINTEXT://<SERVER_IP>:9092
listener.security.protocol.map=CONTROLLER:PLAINTEXT,PLAINTEXT:PLAINTEXT
inter.broker.listener.name=PLAINTEXT

log.dirs=/data/data/kafka
num.partitions=3
offsets.topic.replication.factor=1
transaction.state.log.replication.factor=1
transaction.state.log.min.isr=1
log.retention.hours=168
```

注意：

- `advertised.listeners` 必须写客户端能访问的地址，不能写 `0.0.0.0`。
- 单机环境下副本相关参数必须为 `1`。
- 不同 Kafka 实例的 `node.id`、端口、数据目录不能重复。

### 8.3 格式化并启动

```bash
cd /data/apps/kafka
KAFKA_CLUSTER_ID="$(bin/kafka-storage.sh random-uuid)"
bin/kafka-storage.sh format -t "$KAFKA_CLUSTER_ID" -c config/kraft/server.properties

nohup bin/kafka-server-start.sh config/kraft/server.properties > /data/logs/kafka/kafka.log 2>&1 &
```

验证：

```bash
jps -l | grep kafka
ss -lntp | grep 9092

bin/kafka-topics.sh --bootstrap-server <SERVER_IP>:9092 --create --topic test-topic --partitions 1 --replication-factor 1
bin/kafka-topics.sh --bootstrap-server <SERVER_IP>:9092 --list
```

### 8.4 单机多实例端口建议

| 环境 | Broker 端口 | Controller 端口 | 数据目录 |
| --- | --- | --- | --- |
| dev | 9092 | 9093 | `/data/data/kafka-dev` |
| test | 9094 | 9095 | `/data/data/kafka-test` |
| stress | 9096 | 9097 | `/data/data/kafka-stress` |

复制实例时必须先清空或重新格式化目标数据目录，避免多个实例共用同一个 cluster metadata。

## 9.Nacos

### 9.1 下载并解压

```bash
cd /data/softwares
wget https://github.com/alibaba/nacos/releases/download/2.4.3/nacos-server-2.4.3.tar.gz
sudo tar -zxvf nacos-server-2.4.3.tar.gz -C /data/apps
sudo mkdir -p /data/logs/nacos /data/data/nacos
```

### 9.2 配置外置 MySQL

先导入官方初始化 SQL：

```bash
wget https://raw.githubusercontent.com/alibaba/nacos/2.4.3/distribution/conf/mysql-schema.sql
mysql -uroot -p'<MYSQL_ROOT_PASSWORD>' nacos_config < mysql-schema.sql
```

编辑 `/data/apps/nacos/conf/application.properties`：

```properties
server.port=8848
spring.datasource.platform=mysql
db.num=1
db.url.0=jdbc:mysql://127.0.0.1:3306/nacos_config?characterEncoding=utf8&connectTimeout=1000&socketTimeout=3000&autoReconnect=true&useSSL=false&serverTimezone=Asia/Shanghai
db.user.0=<MYSQL_APP_USER>
db.password.0=<MYSQL_APP_PASSWORD>
nacos.core.auth.enabled=true
nacos.core.auth.server.identity.key=<NACOS_IDENTITY_KEY>
nacos.core.auth.server.identity.value=<NACOS_IDENTITY_VALUE>
nacos.core.auth.plugin.nacos.token.secret.key=<NACOS_TOKEN_SECRET>
```

### 9.3 启动

```bash
cd /data/apps/nacos
sh bin/startup.sh -m standalone
tail -f logs/start.out
```

访问：

```text
http://<SERVER_IP>:8848/nacos
```

生产或共享环境中不要继续使用默认账号密码，首次登录后应立即修改。

## 10.Nginx

### 10.1 安装并启动

```bash
sudo yum install -y nginx
sudo systemctl enable nginx
sudo systemctl start nginx
sudo systemctl status nginx
```

### 10.2 示例反向代理配置

`/etc/nginx/conf.d/<APP_NAME>.conf`：

```nginx
server {
    listen 80;
    server_name <SERVER_IP>;

    location /api/ {
        proxy_pass http://127.0.0.1:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    location / {
        root /data/apps/<APP_NAME>/web;
        index index.html;
        try_files $uri $uri/ /index.html;
    }
}
```

检查并重载：

```bash
sudo nginx -t
sudo systemctl reload nginx
```

## 11.Canal

Canal 常用于订阅 MySQL binlog 并投递到 Kafka。部署前需要确认 MySQL 已开启 binlog。

### 11.1 MySQL binlog 配置

`my.cnf` 增加：

```ini
server-id=1
log-bin=mysql-bin
binlog-format=ROW
binlog-row-image=FULL
expire_logs_days=7
```

创建 Canal 用户：

```sql
CREATE USER 'canal'@'%' IDENTIFIED BY '<CANAL_PASSWORD>';
GRANT SELECT, REPLICATION SLAVE, REPLICATION CLIENT ON *.* TO 'canal'@'%';
FLUSH PRIVILEGES;
```

### 11.2 下载并解压

```bash
cd /data/softwares
wget https://github.com/alibaba/canal/releases/download/canal-1.1.8/canal.deployer-1.1.8.tar.gz
sudo mkdir -p /data/apps/canal
sudo tar -zxvf canal.deployer-1.1.8.tar.gz -C /data/apps/canal
```

### 11.3 instance 配置模板

`/data/apps/canal/conf/<APP_NAME>/instance.properties`：

```properties
canal.instance.mysql.slaveId=1234
canal.instance.master.address=127.0.0.1:3306
canal.instance.dbUsername=canal
canal.instance.dbPassword=<CANAL_PASSWORD>
canal.instance.connectionCharset=UTF-8

canal.instance.filter.regex=<APP_SCHEMA_DEV>\\..*
canal.instance.filter.black.regex=mysql\\.slave_.*

canal.mq.topic=<APP_NAME>-binlog
canal.mq.partition=0
```

`/data/apps/canal/conf/canal.properties` 中开启 Kafka：

```properties
canal.serverMode=kafka
kafka.bootstrap.servers=<SERVER_IP>:9092
canal.destinations=<APP_NAME>
```

启动：

```bash
cd /data/apps/canal
sh bin/startup.sh
tail -f logs/canal/canal.log
```

## 12.防火墙端口

按实际需要开放端口，不建议无差别开放所有中间件端口。

```bash
sudo firewall-cmd --state
sudo firewall-cmd --permanent --add-port=3306/tcp
sudo firewall-cmd --permanent --add-port=6379/tcp
sudo firewall-cmd --permanent --add-port=9092/tcp
sudo firewall-cmd --permanent --add-port=8848/tcp
sudo firewall-cmd --permanent --add-port=80/tcp
sudo firewall-cmd --reload
sudo firewall-cmd --list-ports
```

## 13.后端连接信息模板

建议部署后单独维护一份只在团队内部可见的连接信息，不要把真实密码写进公开仓库。

```yaml
mysql:
  host: <SERVER_IP>
  port: 3306
  username: <MYSQL_APP_USER>
  password: <MYSQL_APP_PASSWORD>
  database: <APP_SCHEMA_DEV>

redis:
  host: <SERVER_IP>
  port: 6379
  password: <REDIS_PASSWORD>

kafka:
  bootstrapServers: <SERVER_IP>:9092

nacos:
  serverAddr: <SERVER_IP>:8848
  username: <NACOS_USERNAME>
  password: <NACOS_PASSWORD>
```

## 14.排障速查

| 现象 | 常见原因 | 处理建议 |
| --- | --- | --- |
| MySQL 客户端连不上 | `bind-address` 只监听本地、防火墙未放行、授权主机不匹配 | 检查 `ss -lntp`、防火墙和 MySQL 用户授权 |
| Redis 远程连不上 | `bind` 限制、防火墙未放行、密码错误 | 检查 `/etc/redis.conf`、`firewall-cmd` 和认证参数 |
| Kafka 客户端连接超时 | `advertised.listeners` 写错 | 改成客户端可访问的 `<SERVER_IP>:<PORT>` |
| Kafka 启动失败 | 数据目录已被其他 cluster 格式化 | 清理测试数据目录后重新 `kafka-storage.sh format` |
| Nacos 启动失败 | MySQL 初始化表缺失或认证参数缺失 | 检查 `application.properties` 和 `nacos_config` 表 |
| Canal 没有投递消息 | MySQL binlog 未开启、表过滤正则不匹配、Kafka 地址错误 | 检查 MySQL binlog、Canal instance 配置和 Kafka topic |
