# 麒麟 V10 Docker 中间件部署教程

> 本文由 Docker 中间件部署记录整理而来，已移除或替换服务器 IP、账号、密码、内部项目名、内部截图链接等敏感信息。实际使用时请将占位符替换为自己的环境值。

## 1.环境基准

本文基于以下环境整理：

- 操作系统：麒麟 V10
- 部署方式：Docker Compose
- CPU 架构：请根据实际服务器选择 x86_64 或 aarch64 镜像/安装包
- 适用场景：开发、测试、演示环境
- 生产说明：生产环境需要补充高可用、备份、监控、权限收敛和安全审计

## 2.部署目标

使用 Docker Compose 在一台服务器上集中部署后端常用中间件：

- MySQL
- Redis
- Kafka
- Nacos
- Nginx
- Canal

适合开发、测试、演示环境。

## 3.占位符说明

| 占位符 | 含义 |
| --- | --- |
| `<SERVER_IP>` | 当前部署服务器 IP |
| `<MYSQL_ROOT_PASSWORD>` | MySQL root 密码 |
| `<MYSQL_APP_USER>` | 后端业务 MySQL 用户 |
| `<MYSQL_APP_PASSWORD>` | 后端业务 MySQL 密码 |
| `<REDIS_PASSWORD>` | Redis 密码 |
| `<NACOS_USERNAME>` | Nacos 用户名 |
| `<NACOS_PASSWORD>` | Nacos 密码 |
| `<CANAL_PASSWORD>` | Canal 连接 MySQL 的密码 |
| `<APP_SCHEMA>` | 业务库名 |
| `<APP_NAME>` | 项目或应用名称 |

## 4.端口规划

| 服务 | 容器端口 | 宿主机端口 | 说明 |
| --- | --- | --- | --- |
| MySQL | 3306 | 3306 | 数据库 |
| Redis | 6379 | 6379 | 缓存 |
| Kafka | 9092 | 9092 | 客户端访问 |
| Kafka Controller | 9093 | 不对外开放 | KRaft 内部控制器 |
| Nacos | 8848 | 8848 | 控制台和配置中心 |
| Nginx | 80 | 80 | 前端静态资源和反向代理 |
| Canal | 11111 | 可选 | Canal TCP 端口 |


## 6.Docker 安装

```bash
sudo yum install -y yum-utils device-mapper-persistent-data lvm2
sudo yum-config-manager --add-repo https://download.docker.com/linux/centos/docker-ce.repo
sudo yum install -y docker-ce docker-ce-cli containerd.io docker-compose-plugin

sudo systemctl enable docker
sudo systemctl start docker
docker version
docker compose version
```

如果镜像拉取不稳定，可以配置镜像加速。示例：

```json
{
  "registry-mirrors": [
    "https://<DOCKER_MIRROR>"
  ],
  "log-driver": "json-file",
  "log-opts": {
    "max-size": "100m",
    "max-file": "3"
  }
}
```

保存为 `/etc/docker/daemon.json` 后重启：

```bash
sudo systemctl daemon-reload
sudo systemctl restart docker
```

## 7.环境变量文件

`.env`：

```env
SERVER_IP=<SERVER_IP>

MYSQL_ROOT_PASSWORD=<MYSQL_ROOT_PASSWORD>
MYSQL_APP_USER=<MYSQL_APP_USER>
MYSQL_APP_PASSWORD=<MYSQL_APP_PASSWORD>
MYSQL_DATABASE=<APP_SCHEMA>

REDIS_PASSWORD=<REDIS_PASSWORD>

NACOS_USERNAME=<NACOS_USERNAME>
NACOS_PASSWORD=<NACOS_PASSWORD>

CANAL_PASSWORD=<CANAL_PASSWORD>
APP_NAME=<APP_NAME>
```

`.env` 中会包含真实密码，建议：

- 不提交到公开仓库。
- 文件权限设置为 `600`。
- 团队共享时使用密码管理工具或受控配置中心。

```bash
chmod 600 .env
```

## 8.Compose 文件

`docker-compose.yml`：

```yaml
services:
  mysql:
    image: mysql:5.7
    container_name: middleware-mysql
    restart: unless-stopped
    environment:
      MYSQL_ROOT_PASSWORD: ${MYSQL_ROOT_PASSWORD}
      MYSQL_DATABASE: ${MYSQL_DATABASE}
      MYSQL_USER: ${MYSQL_APP_USER}
      MYSQL_PASSWORD: ${MYSQL_APP_PASSWORD}
      TZ: Asia/Shanghai
    ports:
      - "3306:3306"
    volumes:
      - ./mysql/data:/var/lib/mysql
      - ./mysql/conf/my.cnf:/etc/mysql/conf.d/my.cnf:ro
      - ./mysql/logs:/var/log/mysql
      - ./mysql/init:/docker-entrypoint-initdb.d:ro
    healthcheck:
      test: ["CMD-SHELL", "mysqladmin ping -uroot -p$${MYSQL_ROOT_PASSWORD} --silent"]
      interval: 10s
      timeout: 5s
      retries: 10

  redis:
    image: redis:7.2
    container_name: middleware-redis
    restart: unless-stopped
    command: ["redis-server", "/usr/local/etc/redis/redis.conf"]
    ports:
      - "6379:6379"
    volumes:
      - ./redis/conf/redis.conf:/usr/local/etc/redis/redis.conf:ro
      - ./redis/data:/data
    healthcheck:
      test: ["CMD-SHELL", "redis-cli -a $${REDIS_PASSWORD} ping | grep PONG"]
      interval: 10s
      timeout: 5s
      retries: 10
    environment:
      REDIS_PASSWORD: ${REDIS_PASSWORD}

  kafka:
    image: bitnami/kafka:3.7
    container_name: middleware-kafka
    restart: unless-stopped
    ports:
      - "9092:9092"
    environment:
      KAFKA_CFG_NODE_ID: 1
      KAFKA_CFG_PROCESS_ROLES: broker,controller
      KAFKA_CFG_CONTROLLER_QUORUM_VOTERS: 1@kafka:9093
      KAFKA_CFG_LISTENERS: PLAINTEXT://:9092,CONTROLLER://:9093
      KAFKA_CFG_ADVERTISED_LISTENERS: PLAINTEXT://${SERVER_IP}:9092
      KAFKA_CFG_LISTENER_SECURITY_PROTOCOL_MAP: CONTROLLER:PLAINTEXT,PLAINTEXT:PLAINTEXT
      KAFKA_CFG_CONTROLLER_LISTENER_NAMES: CONTROLLER
      KAFKA_CFG_INTER_BROKER_LISTENER_NAME: PLAINTEXT
      KAFKA_CFG_OFFSETS_TOPIC_REPLICATION_FACTOR: 1
      KAFKA_CFG_TRANSACTION_STATE_LOG_REPLICATION_FACTOR: 1
      KAFKA_CFG_TRANSACTION_STATE_LOG_MIN_ISR: 1
      ALLOW_PLAINTEXT_LISTENER: "yes"
    volumes:
      - ./kafka/data:/bitnami/kafka
    healthcheck:
      test: ["CMD-SHELL", "kafka-topics.sh --bootstrap-server 127.0.0.1:9092 --list >/dev/null 2>&1"]
      interval: 10s
      timeout: 5s
      retries: 20

  nacos:
    image: nacos/nacos-server:v2.4.3
    container_name: middleware-nacos
    restart: unless-stopped
    depends_on:
      mysql:
        condition: service_healthy
    environment:
      MODE: standalone
      SPRING_DATASOURCE_PLATFORM: mysql
      MYSQL_SERVICE_HOST: mysql
      MYSQL_SERVICE_PORT: 3306
      MYSQL_SERVICE_DB_NAME: nacos_config
      MYSQL_SERVICE_USER: ${MYSQL_APP_USER}
      MYSQL_SERVICE_PASSWORD: ${MYSQL_APP_PASSWORD}
      NACOS_AUTH_ENABLE: "true"
      NACOS_AUTH_IDENTITY_KEY: <NACOS_IDENTITY_KEY>
      NACOS_AUTH_IDENTITY_VALUE: <NACOS_IDENTITY_VALUE>
      NACOS_AUTH_TOKEN: <NACOS_TOKEN_SECRET>
    ports:
      - "8848:8848"
    volumes:
      - ./nacos/logs:/home/nacos/logs
      - ./nacos/data:/home/nacos/data
    healthcheck:
      test: ["CMD-SHELL", "curl -fsS http://127.0.0.1:8848/nacos/actuator/health | grep UP"]
      interval: 15s
      timeout: 5s
      retries: 20

  canal:
    image: canal/canal-server:v1.1.8
    container_name: middleware-canal
    restart: unless-stopped
    depends_on:
      mysql:
        condition: service_healthy
      kafka:
        condition: service_healthy
    ports:
      - "11111:11111"
    volumes:
      - ./canal/conf:/home/admin/canal-server/conf
      - ./canal/logs:/home/admin/canal-server/logs

  nginx:
    image: nginx:1.26
    container_name: middleware-nginx
    restart: unless-stopped
    depends_on:
      nacos:
        condition: service_started
    ports:
      - "80:80"
    volumes:
      - ./nginx/conf.d:/etc/nginx/conf.d:ro
      - ./nginx/html:/usr/share/nginx/html:ro
      - ./nginx/logs:/var/log/nginx
```

说明：

- Kafka 使用 KRaft 模式，不依赖 ZooKeeper。
- `depends_on.condition: service_healthy` 可以避免依赖服务尚未初始化完成时就启动下游服务。
- `KAFKA_CFG_ADVERTISED_LISTENERS` 必须使用客户端可访问的 `<SERVER_IP>`。
- Nacos 使用 MySQL 时，需要提前初始化 `nacos_config` 数据库和表。

## 9.MySQL 配置

`mysql/conf/my.cnf`：

```ini
[mysqld]
character-set-server=utf8mb4
collation-server=utf8mb4_unicode_ci
default-time-zone=+8:00
lower_case_table_names=1
max_connections=1000

server-id=1
log-bin=mysql-bin
binlog-format=ROW
binlog-row-image=FULL
expire_logs_days=7

[client]
default-character-set=utf8mb4
```

`mysql/init/01-init.sql`：

```sql
CREATE DATABASE IF NOT EXISTS nacos_config DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;
CREATE DATABASE IF NOT EXISTS `<APP_SCHEMA>` DEFAULT CHARACTER SET utf8mb4 COLLATE utf8mb4_unicode_ci;

CREATE USER IF NOT EXISTS 'canal'@'%' IDENTIFIED BY '<CANAL_PASSWORD>';
GRANT SELECT, REPLICATION SLAVE, REPLICATION CLIENT ON *.* TO 'canal'@'%';
FLUSH PRIVILEGES;
```

Nacos 表结构建议使用对应版本官方 SQL 初始化，不要手写简化版。

## 10.Redis 配置

`redis/conf/redis.conf`：

```conf
bind 0.0.0.0
protected-mode yes
port 6379
requirepass <REDIS_PASSWORD>
appendonly yes
dir /data
maxmemory 256mb
maxmemory-policy allkeys-lru
```

如果不需要外部访问 Redis，建议不要映射 `6379:6379`，只让同一个 Compose 网络中的服务访问。

## 11.Nginx 配置

`nginx/conf.d/<APP_NAME>.conf`：

```nginx
server {
    listen 80;
    server_name _;

    location /api/ {
        proxy_pass http://host.docker.internal:8080/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
    }

    location / {
        root /usr/share/nginx/html;
        index index.html;
        try_files $uri $uri/ /index.html;
    }
}
```

Linux Docker 默认没有 `host.docker.internal` 时，可以改成宿主机网关地址，或者把后端服务也放进同一个 Compose 网络。

## 12.Canal 配置

`canal/conf/canal.properties` 示例：

```properties
canal.serverMode=kafka
kafka.bootstrap.servers=kafka:9092
canal.destinations=<APP_NAME>
```

`canal/conf/<APP_NAME>/instance.properties` 示例：

```properties
canal.instance.mysql.slaveId=1234
canal.instance.master.address=mysql:3306
canal.instance.dbUsername=canal
canal.instance.dbPassword=<CANAL_PASSWORD>
canal.instance.connectionCharset=UTF-8

canal.instance.filter.regex=<APP_SCHEMA>\\..*
canal.instance.filter.black.regex=mysql\\.slave_.*

canal.mq.topic=<APP_NAME>-binlog
canal.mq.partition=0
```

如果 Canal 日志中出现 H2 或 TSDB 文件只读问题，优先检查挂载目录权限；开发测试环境也可以先关闭 TSDB，确认主链路可用后再补充。

## 13.启动与验证

启动：

```bash
cd /data/middleware
docker compose up -d
docker compose ps
```

查看日志：

```bash
docker compose logs -f mysql
docker compose logs -f kafka
docker compose logs -f nacos
docker compose logs -f canal
```

验证 MySQL：

```bash
docker compose exec mysql mysql -uroot -p'<MYSQL_ROOT_PASSWORD>' -e "SHOW DATABASES;"
```

验证 Redis：

```bash
docker compose exec redis redis-cli -a '<REDIS_PASSWORD>' ping
```

验证 Kafka：

```bash
docker compose exec kafka kafka-topics.sh --bootstrap-server 127.0.0.1:9092 --list
docker compose exec kafka kafka-topics.sh --bootstrap-server 127.0.0.1:9092 --create --topic test-topic --partitions 1 --replication-factor 1
```

验证 Nacos：

```bash
curl -I http://<SERVER_IP>:8848/nacos/
```

## 14.防火墙

按实际需要开放端口：

```bash
sudo firewall-cmd --permanent --add-port=3306/tcp
sudo firewall-cmd --permanent --add-port=6379/tcp
sudo firewall-cmd --permanent --add-port=9092/tcp
sudo firewall-cmd --permanent --add-port=8848/tcp
sudo firewall-cmd --permanent --add-port=80/tcp
sudo firewall-cmd --reload
sudo firewall-cmd --list-ports
```

安全建议：

- MySQL、Redis、Kafka 不一定要对公网或办公网开放。
- 能固定访问来源时，优先用安全组或防火墙限制来源 IP。
- Redis 必须设置强密码。
- Nacos 默认账号密码必须修改。

## 15.常见问题

| 问题 | 原因 | 处理 |
| --- | --- | --- |
| Kafka 容器启动后客户端连不上 | `advertised.listeners` 写成容器名、localhost 或 `0.0.0.0` | 改为客户端可访问的 `<SERVER_IP>:9092` |
| Kafka 数据目录权限错误 | 挂载目录 owner 与镜像运行用户不匹配 | 清理测试数据后调整目录权限，再重建容器 |
| Nacos 一直重启 | MySQL 未就绪、`nacos_config` 未初始化、认证参数缺失 | 先确认 MySQL 健康检查，再导入官方 SQL |
| Canal 无法连接 MySQL | MySQL 未开启 binlog 或 Canal 用户权限不足 | 检查 `log-bin`、`binlog-format=ROW` 和复制权限 |
| Canal 向 Kafka 写入失败 | Kafka 地址不可达或 topic 配置错误 | 在 Canal 容器内测试 `kafka:9092`，检查 topic |
| Docker Hub 拉取失败 | 网络或镜像源不稳定 | 配置可用镜像源，或提前离线导入镜像 |

## 16.维护建议

- Compose 文件可以提交到仓库，但 `.env` 不要提交真实密码。
- 所有中间件数据目录都应纳入备份规划。
- 升级镜像前先在测试环境验证数据兼容性。
- 对 MySQL、Redis、Nacos、Kafka 暴露端口做最小化开放。
- 文档中记录操作步骤，不记录真实凭据。
