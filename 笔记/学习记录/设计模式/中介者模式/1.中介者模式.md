### 中介者模式教程

中介者模式（Mediator Pattern）是一种行为型设计模式。它用一个中介对象封装多个对象之间的交互，使对象之间不再直接相互引用。

### 1. 中介者模式结构

- **Mediator（中介者接口）** ：定义对象之间交互的接口。
- **ConcreteMediator（具体中介者）** ：协调各个同事对象之间的交互。
- **Colleague（同事对象）** ：通过中介者和其他对象通信。

### 2. 中介者模式的优缺点

#### 优点

* **降低对象耦合** ：对象之间不需要直接互相调用。
* **集中管理交互逻辑** ：复杂协作流程更容易维护。
* **简化对象关系** ：多对多关系变成对象到中介者的一对多关系。

#### 缺点

* **中介者可能过大** ：复杂逻辑都集中后容易形成上帝类。
* **调试集中化逻辑** ：所有交互都经过中介者，链路可能不直观。

### 3. 适用场景

* 多个对象之间存在复杂网状依赖。
* 对象交互规则经常变化。
* 希望对象之间只通过统一协调者通信。

### 4. Java 实现示例

下面以聊天室为例。用户之间不直接发送消息，而是通过聊天室中介者转发。

```java
import java.util.ArrayList;
import java.util.List;

interface ChatRoom {
    void send(String message, User sender);
}

class GroupChatRoom implements ChatRoom {
    private final List<User> users = new ArrayList<>();

    public void addUser(User user) {
        users.add(user);
    }

    public void send(String message, User sender) {
        for (User user : users) {
            if (user != sender) {
                user.receive(message);
            }
        }
    }
}

class User {
    private final String name;
    private final ChatRoom chatRoom;

    public User(String name, ChatRoom chatRoom) {
        this.name = name;
        this.chatRoom = chatRoom;
    }

    public void send(String message) {
        System.out.println(name + "发送：" + message);
        chatRoom.send(message, this);
    }

    public void receive(String message) {
        System.out.println(name + "收到：" + message);
    }
}

public class Client {
    public static void main(String[] args) {
        GroupChatRoom room = new GroupChatRoom();
        User zhangsan = new User("张三", room);
        User lisi = new User("李四", room);
        room.addUser(zhangsan);
        room.addUser(lisi);
        zhangsan.send("你好");
    }
}
```

### 5. 注意事项

- 中介者适合复杂交互，不适合简单的一对一调用。
- 中介者不是单纯转发器，重点是封装多个对象之间的交互规则。
- 中介者类要避免无限膨胀，可以按业务场景拆分。
- MVC 中控制器协调视图和模型时，也体现了中介者思想。
