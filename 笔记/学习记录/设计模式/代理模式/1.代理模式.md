### 代理模式教程

代理模式（Proxy Pattern）是一种结构型设计模式。它为真实对象提供一个代理对象，由代理对象控制对真实对象的访问，并可以在访问前后增加控制逻辑。

### 1. 代理模式结构

- **Subject（抽象主题）** ：定义真实对象和代理对象的公共接口。
- **RealSubject（真实主题）** ：真正执行业务逻辑的对象。
- **Proxy（代理）** ：持有真实对象引用，控制访问。
- **Client（客户端）** ：通过抽象主题使用代理对象。

### 2. 代理模式的优缺点

#### 优点

* **控制访问** ：可以增加权限、缓存、日志、延迟加载等逻辑。
* **不修改真实对象** ：横切逻辑集中在代理中。
* **隐藏复杂访问** ：可用于远程代理、虚拟代理、保护代理等场景。

#### 缺点

* **增加代理层** ：调用链路更长。
* **代理逻辑可能膨胀** ：代理承担过多职责时会变复杂。

### 3. 适用场景

* 需要权限控制、日志、缓存、事务等横切逻辑。
* 需要延迟加载开销较大的对象。
* 需要远程代理或保护代理。

### 4. Java 实现示例

下面以文档访问为例。代理对象在访问真实文档前做权限校验。

```java
interface Document {
    void read();
}

class RealDocument implements Document {
    private final String name;

    public RealDocument(String name) {
        this.name = name;
    }

    public void read() {
        System.out.println("读取文档：" + name);
    }
}

class DocumentProxy implements Document {
    private final String name;
    private final boolean admin;
    private RealDocument realDocument;

    public DocumentProxy(String name, boolean admin) {
        this.name = name;
        this.admin = admin;
    }

    public void read() {
        if (!admin) {
            System.out.println("权限不足，无法读取文档");
            return;
        }
        if (realDocument == null) {
            realDocument = new RealDocument(name);
        }
        realDocument.read();
    }
}

public class Client {
    public static void main(String[] args) {
        Document document = new DocumentProxy("系统设计.md", true);
        document.read();
    }
}
```

### 5. 注意事项

- 代理模式和装饰器模式结构相似，但代理重点是控制访问，装饰器重点是增强功能。
- 代理的重点是控制是否、何时、如何访问真实对象；装饰器通常默认保留原调用，并叠加新的职责。
- Spring AOP、事务代理、RPC 客户端代理都体现了代理思想。
- 代理对象最好和真实对象实现同一个接口。
