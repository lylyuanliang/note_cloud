### 原型模式教程

原型模式（Prototype Pattern）是一种创建型设计模式。它通过复制已有对象来创建新对象，而不是通过 `new` 和复杂初始化流程重新创建对象。

当对象创建成本较高、初始化步骤复杂，或者需要在运行时动态决定创建哪一种对象时，可以考虑使用原型模式。

### 1. 原型模式结构

原型模式通常包含以下角色：

- **Prototype（抽象原型）** ：声明克隆自身的方法。
- **ConcretePrototype（具体原型）** ：实现克隆方法，复制当前对象。
- **Client（客户端）** ：通过原型对象复制出新的对象。

### 2. 原型模式的优缺点

#### 优点

* **隐藏创建细节** ：客户端只需要复制已有对象，不需要知道对象如何创建。
* **减少重复初始化成本** ：适合复制初始化成本高的对象。
* **运行时更灵活** ：可以在运行时注册不同原型，再按需克隆。

#### 缺点

* **深拷贝实现复杂** ：对象内部引用层级较深时，需要处理引用对象的复制。
* **容易混淆浅拷贝和深拷贝** ：浅拷贝会共享引用对象，可能导致状态互相影响。
* **克隆逻辑需要维护** ：新增字段时，要确认克隆方法是否也需要调整。

### 3. 适用场景

* 对象创建过程复杂，直接创建成本较高。
* 需要大量创建相似对象，只是部分属性不同。
* 希望避免客户端依赖具体类的构造过程。
* 需要保存某个对象状态，并在此基础上复制新对象。

### 4. Java 实现示例

下面以简历模板为例。不同候选人的简历结构相似，可以先创建一个模板，再复制出具体简历。

#### 4.1 地址类

```java
public class Address implements Cloneable {
    private String city;
    private String detail;

    public Address(String city, String detail) {
        this.city = city;
        this.detail = detail;
    }

    public void setDetail(String detail) {
        this.detail = detail;
    }

    @Override
    public Address clone() {
        try {
            return (Address) super.clone();
        } catch (CloneNotSupportedException e) {
            throw new AssertionError(e);
        }
    }

    @Override
    public String toString() {
        return city + " " + detail;
    }
}
```

#### 4.2 简历类

```java
public class Resume implements Cloneable {
    private String name;
    private String position;
    private Address address;

    public Resume(String name, String position, Address address) {
        this.name = name;
        this.position = position;
        this.address = address;
    }

    public void setName(String name) {
        this.name = name;
    }

    public Address getAddress() {
        return address;
    }

    @Override
    public Resume clone() {
        try {
            Resume copy = (Resume) super.clone();
            copy.address = this.address.clone();
            return copy;
        } catch (CloneNotSupportedException e) {
            throw new AssertionError(e);
        }
    }

    @Override
    public String toString() {
        return "Resume{name='" + name + "', position='" + position + "', address=" + address + "}";
    }
}
```

#### 4.3 客户端使用示例

```java
public class Client {
    public static void main(String[] args) {
        Resume template = new Resume("模板", "Java开发工程师", new Address("北京", "海淀区"));

        Resume resume1 = template.clone();
        resume1.setName("张三");

        Resume resume2 = template.clone();
        resume2.setName("李四");
        resume2.getAddress().setDetail("朝阳区");

        System.out.println(resume1);
        System.out.println(resume2);
    }
}
```

### 5. 浅拷贝和深拷贝

- **浅拷贝** ：只复制对象本身，引用类型字段仍然指向同一个对象。
- **深拷贝** ：不仅复制对象本身，也复制对象内部引用的对象。

Java 的 `Object.clone()` 默认是浅拷贝。如果对象内部有可变引用字段，通常需要手动实现深拷贝。

### 6. 注意事项

- 不要只因为想少写构造函数就使用原型模式。
- 对象内部有集合、数组、其他可变对象时，要明确是否需要深拷贝。
- 在 Java 中使用 `Cloneable` 有一定历史包袱，实际项目中也可以使用复制构造函数、静态工厂方法或序列化方式实现复制。
