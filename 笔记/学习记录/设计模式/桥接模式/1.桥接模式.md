### 桥接模式教程

桥接模式（Bridge Pattern）是一种结构型设计模式。它将抽象部分和实现部分分离，使二者可以独立变化，避免多维度变化导致类数量爆炸。

### 1. 桥接模式结构

- **Abstraction（抽象）** ：定义高层接口，持有实现者引用。
- **RefinedAbstraction（扩展抽象）** ：扩展高层接口。
- **Implementor（实现者）** ：定义实现维度的接口。
- **ConcreteImplementor（具体实现者）** ：实现具体行为。

### 2. 桥接模式的优缺点

#### 优点

* **分离变化维度** ：降低类爆炸风险。
* **独立扩展** ：抽象和实现可以分别扩展。
* **组合优于继承** ：比多层继承更灵活。

#### 缺点

* **结构更复杂** ：比普通继承更难理解。
* **需要识别变化维度** ：设计前需要判断哪些维度会独立变化。

### 3. 适用场景

* 一个对象存在两个或多个独立变化维度。
* 不希望通过多层继承组合所有变化。
* 希望通过组合选择或替换实现，必要时可在运行时切换。

### 4. Java 实现示例

下面以消息发送为例。消息类型和发送渠道是两个变化维度，可以用桥接模式拆开。

```java
interface MessageSender {
    void send(String content);
}

class EmailSender implements MessageSender {
    public void send(String content) {
        System.out.println("邮件发送：" + content);
    }
}

class SmsSender implements MessageSender {
    public void send(String content) {
        System.out.println("短信发送：" + content);
    }
}

abstract class Message {
    protected MessageSender sender;

    protected Message(MessageSender sender) {
        this.sender = sender;
    }

    public abstract void send(String content);
}

class UrgentMessage extends Message {
    public UrgentMessage(MessageSender sender) {
        super(sender);
    }

    public void send(String content) {
        sender.send("[紧急] " + content);
    }
}

class NormalMessage extends Message {
    public NormalMessage(MessageSender sender) {
        super(sender);
    }

    public void send(String content) {
        sender.send(content);
    }
}

public class Client {
    public static void main(String[] args) {
        Message urgentSms = new UrgentMessage(new SmsSender());
        urgentSms.send("服务器磁盘空间不足");

        Message normalEmail = new NormalMessage(new EmailSender());
        normalEmail.send("日报生成完成");
    }
}
```

### 5. 注意事项

- 桥接模式重点是拆分独立变化维度。
- 如果只有一个变化维度，普通继承或组合可能更简单。
- 桥接模式和适配器模式都使用组合，但桥接偏设计阶段，适配器偏兼容已有接口。
