### 状态模式教程

状态模式（State Pattern）是一种行为型设计模式。它允许对象在内部状态改变时改变行为，看起来就像对象的类发生了变化。

### 1. 状态模式结构

- **Context（上下文）** ：持有当前状态，并把行为委托给状态对象。
- **State（状态接口）** ：定义状态相关行为。
- **ConcreteState（具体状态）** ：实现某个状态下的行为，并可切换上下文状态。

### 2. 状态模式的优缺点

#### 优点

* **消除复杂条件判断** ：把不同状态行为拆分到独立状态类。
* **状态职责清晰** ：每个状态类只处理本状态下的行为。
* **便于扩展新状态** ：新增状态类比修改大量 if-else 更安全。

#### 缺点

* **类数量增加** ：每个状态通常对应一个类。
* **状态切换分散** ：切换逻辑需要清晰设计，否则难以追踪。

### 3. 适用场景

* 对象行为依赖状态，并且状态较多。
* 代码中存在大量和状态相关的条件分支。
* 状态转换规则明确。

### 4. Java 实现示例

```java
interface OrderState {
    void pay(OrderContext context);
    void ship(OrderContext context);
}

class UnpaidState implements OrderState {
    public void pay(OrderContext context) {
        System.out.println("支付成功");
        context.setState(new PaidState());
    }

    public void ship(OrderContext context) {
        System.out.println("未支付，不能发货");
    }
}

class PaidState implements OrderState {
    public void pay(OrderContext context) {
        System.out.println("订单已支付，无需重复支付");
    }

    public void ship(OrderContext context) {
        System.out.println("发货成功");
    }
}

class OrderContext {
    private OrderState state = new UnpaidState();

    public void setState(OrderState state) {
        this.state = state;
    }

    public void pay() {
        state.pay(this);
    }

    public void ship() {
        state.ship(this);
    }
}

public class Client {
    public static void main(String[] args) {
        OrderContext order = new OrderContext();
        order.ship();
        order.pay();
        order.ship();
    }
}
```

### 5. 注意事项

- 状态模式适合状态和行为强相关的对象。
- 如果只有两三个简单状态，普通条件判断可能更直观。
- 状态模式关注“同一对象在不同状态下行为不同”，策略模式关注“算法可替换”。
