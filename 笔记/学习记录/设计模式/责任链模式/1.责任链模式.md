### 责任链模式教程

责任链模式（Chain of Responsibility Pattern）是一种行为型设计模式。它将多个处理对象连接成一条链，请求沿着链传递，直到被某个处理器处理或链路结束。

### 1. 责任链模式结构

- **Handler（抽象处理器）** ：定义处理请求的方法，并持有下一个处理器。
- **ConcreteHandler（具体处理器）** ：判断自己是否能处理请求，不能处理则交给下一个处理器。
- **Client（客户端）** ：组装责任链并提交请求。

### 2. 责任链模式的优缺点

#### 优点

* **降低耦合** ：请求发送者不需要知道具体由谁处理。
* **灵活扩展** ：可以动态增加、删除或调整处理器顺序。
* **职责清晰** ：每个处理器只关注自己能处理的部分。

#### 缺点

* **请求可能无人处理** ：链路设计不当会导致请求被漏掉。
* **调试不直观** ：链路较长时难以快速定位处理节点。

### 3. 适用场景

* 多个对象都有机会处理同一个请求。
* 多个处理器都有机会处理请求，但请求通常由其中一个处理器接收。
* 需要动态调整处理顺序，例如审批流、异常处理链。

### 4. Java 实现示例

```java
abstract class Approver {
    protected Approver next;

    public Approver setNext(Approver next) {
        this.next = next;
        return next;
    }

    public abstract void approve(int amount);

    protected void approveNext(int amount) {
        if (next != null) {
            next.approve(amount);
            return;
        }
        System.out.println("无人审批：" + amount);
    }
}

class Manager extends Approver {
    public void approve(int amount) {
        if (amount <= 1000) {
            System.out.println("经理审批通过：" + amount);
        } else {
            approveNext(amount);
        }
    }
}

class Director extends Approver {
    public void approve(int amount) {
        if (amount <= 5000) {
            System.out.println("总监审批通过：" + amount);
        } else {
            approveNext(amount);
        }
    }
}

public class Client {
    public static void main(String[] args) {
        Approver manager = new Manager();
        manager.setNext(new Director());
        manager.approve(3000);
        manager.approve(8000);
    }
}
```

### 5. 注意事项

- 要明确链路结束时如何处理无人接收的请求。
- 经典责任链通常是“某个节点处理后停止”；过滤器链更像“多个节点依次处理”的变体。
- 处理器顺序会影响结果，需要统一管理。
- Servlet Filter、Spring Security 过滤器链都体现了责任链思想。
