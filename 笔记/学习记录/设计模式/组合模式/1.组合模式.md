### 组合模式教程

组合模式（Composite Pattern）是一种结构型设计模式。它将对象组织成树形结构，用于表示“部分-整体”层次，并让客户端以一致方式处理单个对象和组合对象。

### 1. 组合模式结构

- **Component（抽象组件）** ：定义叶子和组合对象的公共接口。
- **Leaf（叶子节点）** ：没有子节点的对象。
- **Composite（组合节点）** ：可以包含子组件的对象。
- **Client（客户端）** ：通过组件接口统一使用对象。

### 2. 组合模式的优缺点

#### 优点

* **统一处理** ：客户端可以一致处理单个对象和组合对象。
* **适合树形结构** ：天然适合菜单、目录、组织架构等层次关系。
* **易于扩展节点类型** ：新增组件类型不影响客户端遍历逻辑。

#### 缺点

* **接口可能过宽** ：叶子节点可能拥有不适用的方法。
* **不适合非层次结构** ：对象之间没有整体和部分关系时不适用。

### 3. 适用场景

* 菜单、组织架构、文件目录等树形结构。
* 希望统一处理容器对象和单个对象。
* 需要递归遍历或聚合计算。

### 4. Java 实现示例

```java
import java.util.ArrayList;
import java.util.List;

interface MenuComponent {
    void print();
}

class MenuItem implements MenuComponent {
    private final String name;

    public MenuItem(String name) {
        this.name = name;
    }

    public void print() {
        System.out.println("菜单项：" + name);
    }
}

class MenuGroup implements MenuComponent {
    private final String name;
    private final List<MenuComponent> children = new ArrayList<>();

    public MenuGroup(String name) {
        this.name = name;
    }

    public void add(MenuComponent component) {
        children.add(component);
    }

    public void print() {
        System.out.println("菜单组：" + name);
        for (MenuComponent child : children) {
            child.print();
        }
    }
}

public class Client {
    public static void main(String[] args) {
        MenuGroup root = new MenuGroup("系统管理");
        root.add(new MenuItem("用户管理"));
        root.add(new MenuItem("角色管理"));
        root.print();
    }
}
```

### 5. 注意事项

- 组合模式适合天然树形结构。
- 公共接口要谨慎设计，避免叶子节点出现无意义操作。
- 组合模式常和迭代器、访问者模式一起使用。
