### 迭代器模式教程

迭代器模式（Iterator Pattern）是一种行为型设计模式。它提供一种顺序访问聚合对象内部元素的方法，而不暴露聚合对象的内部表示。

### 1. 迭代器模式结构

- **Iterator（迭代器接口）** ：定义 `hasNext()`、`next()` 等遍历方法。
- **ConcreteIterator（具体迭代器）** ：实现具体遍历逻辑。
- **Aggregate（聚合接口）** ：声明创建迭代器的方法。
- **ConcreteAggregate（具体聚合）** ：保存元素并返回迭代器。

### 2. 迭代器模式的优缺点

#### 优点

* **封装集合结构** ：客户端不需要知道内部存储方式。
* **统一遍历方式** ：不同集合可以提供一致的访问接口。
* **支持多种遍历策略** ：可以提供正序、倒序、过滤等迭代器。

#### 缺点

* **简单集合不必手写** ：Java 集合框架已经提供迭代器。
* **迭代过程中修改集合需要谨慎** ：可能出现并发修改问题。

### 3. 适用场景

* 需要遍历聚合对象但不暴露内部结构。
* 需要为同一集合提供多种遍历方式。
* 需要统一不同集合类型的访问方式。

### 4. Java 实现示例

```java
import java.util.ArrayList;
import java.util.List;
import java.util.NoSuchElementException;

interface NameIterator {
    boolean hasNext();
    String next();
}

class ListNameIterator implements NameIterator {
    private final List<String> names;
    private int index = 0;

    public ListNameIterator(List<String> names) {
        this.names = names;
    }

    public boolean hasNext() {
        return index < names.size();
    }

    public String next() {
        if (!hasNext()) {
            throw new NoSuchElementException();
        }
        return names.get(index++);
    }
}

class NameCollection {
    private final List<String> names = new ArrayList<>();

    public void add(String name) {
        names.add(name);
    }

    public NameIterator iterator() {
        return new ListNameIterator(names);
    }
}

public class Client {
    public static void main(String[] args) {
        NameCollection collection = new NameCollection();
        collection.add("张三");
        collection.add("李四");

        NameIterator iterator = collection.iterator();
        while (iterator.hasNext()) {
            System.out.println(iterator.next());
        }
    }
}
```

### 5. 注意事项

- Java 中通常直接使用 `java.util.Iterator` 和增强 `for`。
- 迭代器模式重点是分离集合存储和遍历行为。
- 如果集合结构变化频繁，要注意迭代器失效问题。
