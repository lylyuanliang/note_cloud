### 访问者模式教程

访问者模式（Visitor Pattern）是一种行为型设计模式。它把作用于对象结构中各元素的操作封装到访问者中，使得在不修改元素类的情况下增加新操作。

### 1. 访问者模式结构

- **Visitor（访问者接口）** ：为每种元素定义访问方法。
- **ConcreteVisitor（具体访问者）** ：实现具体操作。
- **Element（元素接口）** ：定义接受访问者的方法。
- **ConcreteElement（具体元素）** ：调用访问者访问自己。
- **ObjectStructure（对象结构）** ：保存元素集合并提供遍历入口。

### 2. 访问者模式的优缺点

#### 优点

* **新增操作方便** ：新增访问者即可增加一类操作。
* **操作集中管理** ：同一操作对不同元素的逻辑集中在访问者中。
* **适合稳定对象结构** ：元素类型稳定时扩展操作很方便。

#### 缺点

* **新增元素困难** ：新增元素类型需要修改所有访问者。
* **破坏封装风险** ：访问者可能需要元素暴露更多内部信息。
* **结构复杂** ：双分派概念理解成本较高。

### 3. 适用场景

* 对象结构稳定，但经常需要增加新操作。
* 需要对一组不同类型对象执行不同逻辑。
* 编译器 AST、报表导出、对象结构统计等场景。

### 4. Java 实现示例

```java
import java.util.ArrayList;
import java.util.List;

interface Visitor {
    void visit(Book book);
    void visit(Fruit fruit);
}

interface Product {
    void accept(Visitor visitor);
}

class Book implements Product {
    private final int price;

    public Book(int price) {
        this.price = price;
    }

    public int getPrice() {
        return price;
    }

    public void accept(Visitor visitor) {
        visitor.visit(this);
    }
}

class Fruit implements Product {
    private final int price;

    public Fruit(int price) {
        this.price = price;
    }

    public int getPrice() {
        return price;
    }

    public void accept(Visitor visitor) {
        visitor.visit(this);
    }
}

class DiscountVisitor implements Visitor {
    public void visit(Book book) {
        System.out.println("图书折后价：" + (book.getPrice() - 10));
    }

    public void visit(Fruit fruit) {
        System.out.println("水果折后价：" + (int) (fruit.getPrice() * 0.9));
    }
}

class ProductCatalog {
    private final List<Product> products = new ArrayList<>();

    public void add(Product product) {
        products.add(product);
    }

    public void accept(Visitor visitor) {
        for (Product product : products) {
            product.accept(visitor);
        }
    }
}

public class Client {
    public static void main(String[] args) {
        ProductCatalog catalog = new ProductCatalog();
        catalog.add(new Book(100));
        catalog.add(new Fruit(20));

        Visitor visitor = new DiscountVisitor();
        catalog.accept(visitor);
    }
}
```

### 5. 注意事项

- 访问者模式适合元素类型稳定、操作变化频繁的结构。
- 如果元素类型经常变化，访问者模式维护成本会很高。
- 访问者模式核心是双分派：元素决定调用访问者的哪个重载方法。
