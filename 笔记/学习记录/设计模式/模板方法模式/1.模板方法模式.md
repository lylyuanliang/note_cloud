### 模板方法模式教程

模板方法模式（Template Method Pattern）是一种行为型设计模式。它在父类中定义算法骨架，把某些步骤延迟到子类实现。

### 1. 模板方法模式结构

- **AbstractClass（抽象类）** ：定义模板方法和算法步骤。
- **ConcreteClass（具体类）** ：实现或覆盖部分步骤。
- **Template Method（模板方法）** ：固定算法执行顺序，通常声明为 `final`。
- **Hook Method（钩子方法）** ：提供默认实现，子类可以按需覆盖。

### 2. 模板方法模式的优缺点

#### 优点

* **复用算法骨架** ：公共流程集中在父类。
* **控制执行顺序** ：子类只实现变化步骤。
* **符合开闭原则** ：新增流程变体可以增加子类。

#### 缺点

* **依赖继承** ：父类变化可能影响所有子类。
* **子类数量增加** ：不同变体通常需要不同子类。
* **反向控制不直观** ：父类调用子类方法，新手理解有成本。

### 3. 适用场景

* 多个类有相同流程，但部分步骤不同。
* 需要固定算法顺序，防止子类改变流程。
* 框架定义流程，业务代码实现扩展点。

### 4. Java 实现示例

```java
abstract class DataImporter {
    public final void importData() {
        readData();
        validateData();
        beforeSave();
        saveData();
    }

    protected abstract void readData();

    protected void validateData() {
        System.out.println("校验数据");
    }

    protected void beforeSave() {
        // 钩子方法：默认什么都不做，子类可以选择覆盖
    }

    protected abstract void saveData();
}

class CsvDataImporter extends DataImporter {
    protected void readData() {
        System.out.println("读取 CSV 数据");
    }

    protected void beforeSave() {
        System.out.println("保存前清理空行");
    }

    protected void saveData() {
        System.out.println("保存 CSV 数据");
    }
}

public class Client {
    public static void main(String[] args) {
        DataImporter importer = new CsvDataImporter();
        importer.importData();
    }
}
```

### 5. 注意事项

- 模板方法适合流程稳定、步骤可变的场景。
- 模板方法通常用 `final` 修饰，避免子类破坏流程。
- 抽象方法表示子类必须实现的步骤，钩子方法表示子类可选覆盖的扩展点。
- Spring、Servlet 等框架中大量使用模板方法思想。
