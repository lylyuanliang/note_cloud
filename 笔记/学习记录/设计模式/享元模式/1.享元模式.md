### 享元模式教程

享元模式（Flyweight Pattern）是一种结构型设计模式。它通过共享包含内部状态的细粒度对象，并把外部状态从对象中剥离出来，减少对象数量和内存开销。

### 1. 享元模式结构

- **Flyweight（享元接口）** ：定义共享对象的公共行为。
- **ConcreteFlyweight（具体享元）** ：保存可共享的内部状态。
- **FlyweightFactory（享元工厂）** ：缓存和管理享元对象。
- **Client（客户端）** ：提供外部状态并使用享元对象。

### 2. 享元模式的优缺点

#### 优点

* **节省内存** ：减少重复对象数量。
* **提高复用** ：共享对象内部状态。
* **适合大量相似对象** ：对象规模越大收益越明显。

#### 缺点

* **需要拆分状态** ：必须区分内部状态和外部状态。
* **增加复杂度** ：外部状态由客户端维护，使用不当会出错。

### 3. 适用场景

* 系统中存在大量相似对象。
* 对象大部分状态可以共享。
* 内存占用成为明显问题。

### 4. Java 实现示例

下面以棋子为例。棋子的颜色可以共享，位置由客户端传入。

```java
import java.util.HashMap;
import java.util.Map;

class ChessPiece {
    private final String color;

    public ChessPiece(String color) {
        this.color = color;
    }

    public void display(int x, int y) {
        System.out.println(color + "棋子位置：(" + x + "," + y + ")");
    }
}

class ChessPieceFactory {
    private static final Map<String, ChessPiece> CACHE = new HashMap<>();

    public static ChessPiece getChessPiece(String color) {
        return CACHE.computeIfAbsent(color, ChessPiece::new);
    }
}

public class Client {
    public static void main(String[] args) {
        ChessPiece black1 = ChessPieceFactory.getChessPiece("黑色");
        ChessPiece black2 = ChessPieceFactory.getChessPiece("黑色");
        black1.display(1, 2);
        black2.display(3, 4);
        System.out.println(black1 == black2);
    }
}
```

### 5. 注意事项

- 内部状态是可共享状态，外部状态是每次使用时变化的状态。
- 共享对象应尽量保持不可变；多线程场景下工厂缓存可考虑使用 `ConcurrentHashMap`。
- 享元模式只有在对象数量很大时才值得引入。
- 字符串常量池、Integer 缓存都体现了类似思想。
