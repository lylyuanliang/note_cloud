### 命令模式教程

命令模式（Command Pattern）是一种行为型设计模式。它将一个请求封装成一个命令对象，使请求的发起者和执行者解耦。

### 1. 命令模式结构

- **Command（命令接口）** ：声明执行命令的方法。
- **ConcreteCommand（具体命令）** ：封装接收者和具体操作。
- **Receiver（接收者）** ：真正执行业务动作的对象。
- **Invoker（调用者）** ：持有命令并触发执行。
- **Client（客户端）** ：创建命令并设置调用关系。

### 2. 命令模式的优缺点

#### 优点

* **解耦请求和执行** ：调用者不需要知道具体执行逻辑。
* **支持扩展** ：新增命令不影响已有调用者。
* **便于排队和撤销** ：命令对象可以被保存、记录和重放。

#### 缺点

* **类数量增加** ：每个操作可能都需要一个命令类。
* **简单场景显得繁琐** ：普通方法调用可能更清晰。

### 3. 适用场景

* 需要把请求排队、记录日志或延迟执行。
* 需要支持撤销、重做。
* 需要将调用者和执行者解耦。

### 4. Java 实现示例

```java
interface Command {
    void execute();
}

class Light {
    public void on() {
        System.out.println("打开灯");
    }

    public void off() {
        System.out.println("关闭灯");
    }
}

class LightOnCommand implements Command {
    private final Light light;

    public LightOnCommand(Light light) {
        this.light = light;
    }

    public void execute() {
        light.on();
    }
}

class RemoteControl {
    private Command command;

    public void setCommand(Command command) {
        this.command = command;
    }

    public void pressButton() {
        command.execute();
    }
}

public class Client {
    public static void main(String[] args) {
        Light light = new Light();
        Command command = new LightOnCommand(light);
        RemoteControl remote = new RemoteControl();
        remote.setCommand(command);
        remote.pressButton();
    }
}
```

### 5. 注意事项

- 命令模式适合“请求需要对象化”的场景。
- 撤销功能通常需要命令保存执行前状态，或提供反向命令。
- GUI 按钮事件、任务队列、事务脚本都可以使用命令思想。
