### 装饰器模式教程

装饰器模式（Decorator Pattern）是一种结构型设计模式。它通过包装对象的方式，在不修改原类的情况下动态增强对象功能。

### 1. 装饰器模式结构

- **Component（抽象组件）** ：定义原对象和装饰器的公共接口。
- **ConcreteComponent（具体组件）** ：被装饰的原始对象。
- **Decorator（抽象装饰器）** ：持有组件引用，并实现组件接口。
- **ConcreteDecorator（具体装饰器）** ：在调用原功能前后增加额外行为。

### 2. 装饰器模式的优缺点

#### 优点

* **动态增强功能** ：比继承更灵活。
* **符合开闭原则** ：新增功能通过新增装饰器实现。
* **避免类爆炸** ：可以按需组合多个装饰器。

#### 缺点

* **对象数量增加** ：会产生较多小对象。
* **调试复杂** ：多层装饰后调用链路会变长。

### 3. 适用场景

* 需要给对象动态增加职责。
* 功能可以按层叠方式组合。
* 不希望通过继承创建大量功能组合子类。

### 4. Java 实现示例

下面以咖啡为例。基础咖啡可以被牛奶、糖等装饰器增强。

```java
interface Coffee {
    String getDescription();
    int cost();
}

class SimpleCoffee implements Coffee {
    public String getDescription() {
        return "咖啡";
    }

    public int cost() {
        return 10;
    }
}

abstract class CoffeeDecorator implements Coffee {
    protected final Coffee coffee;

    protected CoffeeDecorator(Coffee coffee) {
        this.coffee = coffee;
    }
}

class MilkDecorator extends CoffeeDecorator {
    public MilkDecorator(Coffee coffee) {
        super(coffee);
    }

    public String getDescription() {
        return coffee.getDescription() + " + 牛奶";
    }

    public int cost() {
        return coffee.cost() + 3;
    }
}

public class Client {
    public static void main(String[] args) {
        Coffee coffee = new MilkDecorator(new SimpleCoffee());
        System.out.println(coffee.getDescription() + "，价格：" + coffee.cost());
    }
}
```

### 5. 注意事项

- 装饰器强调增强单个对象，而不是改变整个类。
- 装饰器和代理结构相似，但意图不同：装饰器增强功能，代理控制访问。
- Java IO 中大量使用了装饰器思想。
