### 解释器模式教程

解释器模式（Interpreter Pattern）是一种行为型设计模式。它为某种语言或表达式定义语法表示，并提供解释执行这些表达式的方式。

### 1. 解释器模式结构

- **Expression（表达式接口）** ：声明解释方法。
- **TerminalExpression（终结符表达式）** ：处理不可再拆分的语法单元。
- **NonterminalExpression（非终结符表达式）** ：组合其他表达式并解释复杂规则。
- **Context（上下文）** ：保存解释过程需要的外部信息。

### 2. 解释器模式的优缺点

#### 优点

* **语法规则清晰** ：每条规则可以对应一个表达式类。
* **易于扩展语法** ：新增规则可新增表达式类。
* **适合小型 DSL** ：可以表达特定领域规则。

#### 缺点

* **类数量多** ：复杂语法会产生大量表达式类。
* **性能一般** ：解释执行通常不适合高性能复杂语言。
* **维护成本高** ：语法复杂时建议使用成熟解析器。

### 3. 适用场景

* 语法简单且稳定。
* 需要解释业务规则、表达式或小型 DSL。
* 规则可以抽象成语法树。

### 4. Java 实现示例

下面实现一个简单权限表达式，支持判断用户是否同时拥有 `ADMIN` 和 `AUDITOR` 两个角色。这里直接构建表达式树，省略字符串解析过程。

```java
import java.util.Arrays;
import java.util.HashSet;
import java.util.Set;

interface Expression {
    boolean interpret(Set<String> roles);
}

class RoleExpression implements Expression {
    private final String role;

    public RoleExpression(String role) {
        this.role = role;
    }

    public boolean interpret(Set<String> roles) {
        return roles.contains(role);
    }
}

class AndExpression implements Expression {
    private final Expression left;
    private final Expression right;

    public AndExpression(Expression left, Expression right) {
        this.left = left;
        this.right = right;
    }

    public boolean interpret(Set<String> roles) {
        return left.interpret(roles) && right.interpret(roles);
    }
}

public class Client {
    public static void main(String[] args) {
        // 相当于解析后的表达式：ADMIN && AUDITOR
        Expression expression = new AndExpression(
                new RoleExpression("ADMIN"),
                new RoleExpression("AUDITOR")
        );
        Set<String> roles = new HashSet<>(Arrays.asList("ADMIN", "AUDITOR"));
        System.out.println(expression.interpret(roles));
    }
}
```

### 5. 注意事项

- 解释器模式不适合复杂通用语言。
- 解析器负责把字符串转成表达式树，解释器负责执行表达式树；两者不是同一个职责。
- 表达式规则多时，可以考虑 ANTLR、规则引擎或表达式引擎。
- 它的核心是把语法规则对象化。
