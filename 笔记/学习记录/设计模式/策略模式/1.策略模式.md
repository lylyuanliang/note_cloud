### 策略模式教程

策略模式（Strategy Pattern）是一种行为型设计模式。它定义一组算法，将每个算法封装起来，并让它们可以相互替换。

### 1. 策略模式结构

- **Strategy（策略接口）** ：定义算法接口。
- **ConcreteStrategy（具体策略）** ：实现具体算法。
- **Context（上下文）** ：持有策略对象，并把算法执行委托给策略。

### 2. 策略模式的优缺点

#### 优点

* **消除条件分支** ：用多态替代大量 if-else。
* **算法可替换** ：运行时可以切换不同策略。
* **符合开闭原则** ：新增策略不需要修改上下文。

#### 缺点

* **策略类数量增加** ：每种算法通常对应一个类。
* **客户端需要了解策略差异** ：需要知道何时选择哪种策略。

### 3. 适用场景

* 有多种算法可以完成同一类任务。
* 算法需要独立扩展或替换。
* 代码中存在大量按类型选择算法的条件分支。

### 4. Java 实现示例

```java
interface DiscountStrategy {
    int discount(int price);
}

class NormalDiscount implements DiscountStrategy {
    public int discount(int price) {
        return price;
    }
}

class VipDiscount implements DiscountStrategy {
    public int discount(int price) {
        return (int) (price * 0.8);
    }
}

class Order {
    private DiscountStrategy discountStrategy;

    public Order(DiscountStrategy discountStrategy) {
        this.discountStrategy = discountStrategy;
    }

    public int calculatePrice(int price) {
        return discountStrategy.discount(price);
    }
}

public class Client {
    public static void main(String[] args) {
        Order order = new Order(new VipDiscount());
        System.out.println(order.calculatePrice(100));
    }
}
```

### 5. 注意事项

- 策略模式关注算法替换，不负责保存对象状态变化。
- 策略选择逻辑可以放在工厂、配置或依赖注入容器中。
- 状态模式和策略模式结构相似，但意图不同：状态模式根据内部状态改变行为，策略模式由外部选择算法。
