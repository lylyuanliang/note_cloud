### 备忘录模式教程

备忘录模式（Memento Pattern）是一种行为型设计模式。它在不破坏对象封装的前提下，保存对象某一时刻的内部状态，并在需要时恢复。

### 1. 备忘录模式结构

- **Originator（发起人）** ：需要保存和恢复状态的对象。
- **Memento（备忘录）** ：保存发起人的内部状态。
- **Caretaker（管理者）** ：保存备忘录，但不直接修改备忘录内容。

### 2. 备忘录模式的优缺点

#### 优点

* **不破坏封装** ：状态由对象自己导出和恢复。
* **支持撤销恢复** ：适合实现撤销、快照和回滚。
* **职责清晰** ：状态保存和业务对象分离。

#### 缺点

* **占用内存** ：保存大量快照可能消耗较多资源。
* **状态过大时成本高** ：需要考虑增量保存或限制历史数量。

### 3. 适用场景

* 需要撤销、重做、回滚功能。
* 需要保存对象历史状态。
* 不希望外部对象直接访问内部状态。

### 4. Java 实现示例

```java
import java.util.Stack;

class Editor {
    private String content;

    public void setContent(String content) {
        this.content = content;
    }

    public Memento save() {
        return new Memento(content);
    }

    public void restore(Memento memento) {
        this.content = memento.getContent();
    }

    public void print() {
        System.out.println(content);
    }
}

class Memento {
    private final String content;

    public Memento(String content) {
        this.content = content;
    }

    String getContent() {
        return content;
    }
}

class History {
    private final Stack<Memento> stack = new Stack<>();

    public void push(Memento memento) {
        stack.push(memento);
    }

    public Memento pop() {
        return stack.pop();
    }
}

public class Client {
    public static void main(String[] args) {
        Editor editor = new Editor();
        History history = new History();

        editor.setContent("第一版");
        history.push(editor.save());
        editor.setContent("第二版");
        editor.restore(history.pop());
        editor.print();
    }
}
```

### 5. 注意事项

- 备忘录对象应尽量保持不可变。
- 快照数量要控制，避免内存占用过高。
- 文本编辑器撤销、游戏存档、事务回滚都可以使用该思想。
